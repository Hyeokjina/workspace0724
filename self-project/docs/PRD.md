- **제품명**: GymHub
- **버전**: 1.0.0
- **작성일**: 2025-01-15
- **최종 수정일**: 2025-01-15
- **작성자**: GymHub 개발팀

---

## 1. 제품 개요 (Product Overview)

### 1.1 제품 소개
GymHub는 헬스장 운영에 필요한 모든 기능을 통합 관리할 수 있는 웹 기반 종합 관리 플랫폼입니다. 회원, 트레이너, 헬스장 운영자 각각의 역할에 맞는 대시보드와 기능을 제공하여 헬스장 운영 효율을 극대화하고 회원 만족도를 향상시킵니다.

### 1.2 배경 및 필요성
- 헬스장 운영자는 회원 관리, 매출 관리, 재고 관리 등 다양한 업무를 수동으로 처리하여 비효율적
- 회원들은 PT 예약, 출결 확인, 헬스장 정보 조회 등을 위해 별도의 앱이나 전화 문의 필요
- 트레이너는 PT 일정 관리 및 회원 관리를 수기로 작성하여 불편함
- 통합 관리 시스템의 부재로 인한 운영 비용 증가 및 고객 불만 증가

### 1.3 제품 목표
1. **운영 효율성 향상**: 헬스장 운영에 필요한 모든 기능을 하나의 플랫폼에서 통합 관리
2. **사용자 경험 개선**: 회원, 트레이너, 운영자 각각의 역할에 최적화된 인터페이스 제공
3. **데이터 기반 의사결정**: 매출 통계, 출결 통계 등 데이터 시각화를 통한 경영 인사이트 제공
4. **접근성 향상**: 웹 기반 플랫폼으로 언제 어디서나 접근 가능

## 2. 목표 및 비전 (Goals & Vision)

### 2.1 비전
"대한민국의 모든 헬스장이 GymHub를 통해 효율적으로 운영되고, 모든 회원이 편리하게 헬스장을 이용하는 생태계 구축"

### 2.2 사업 목표
- **단기 목표 (6개월)**: 파일럿 헬스장 10곳 확보 및 서비스 안정화
- **중기 목표 (1년)**: 전국 100개 헬스장 서비스 제공 및 기능 고도화
- **장기 목표 (3년)**: 헬스장 통합 플랫폼 시장 점유율 1위 달성

### 2.3 주요 성과 지표 (KPI)
1. **사용자 증가율**: 월평균 활성 사용자 수 30% 증가
2. **고객 만족도**: NPS(Net Promoter Score) 70 이상
3. **운영 효율성**: 헬스장 운영 시간 30% 절감
4. **매출 증대**: 제휴 헬스장 평균 매출 20% 증가

## 3. 대상 사용자 (Target Users)

### 3.1 사용자 페르소나

#### 페르소나 1: 헬스장 운영자 (김영희, 35세)
- **직업**: 헬스장 원장
- **목표**: 효율적인 회원 관리 및 매출 증대
- **불편사항**: 수동 회원 관리로 인한 업무 과중, 실시간 통계 부재
- **기대 효과**: 자동화된 관리 시스템으로 업무 시간 절감, 데이터 기반 경영

#### 페르소나 2: 일반 회원 (박민수, 28세)
- **직업**: 직장인
- **목표**: 편리한 헬스장 이용 및 운동 목표 달성
- **불편사항**: PT 예약 절차 복잡, 출결 확인 어려움
- **기대 효과**: 간편한 예약 시스템, 운동 기록 관리

#### 페르소나 3: 트레이너 (이준호, 30세)
- **직업**: 퍼스널 트레이너
- **목표**: 체계적인 회원 관리 및 PT 일정 관리
- **불편사항**: 수기 일정 관리로 인한 실수 발생
- **기대 효과**: 디지털 일정 관리, 회원 정보 통합 조회

### 3.2 사용자 역할별 권한

| 역할 | 권한 레벨 | 주요 기능 |
|------|-----------|-----------|
| **일반회원** | Level 1 | 회원권 구매, PT 예약, 출결 조회, 인바디 기록 |
| **트레이너** | Level 2 | PT 일정 관리, 회원 예약 승인, 담당 회원 관리 |
| **헬스장 운영자** | Level 3 | 모든 관리 기능 (회원, 트레이너, 매출, 재고 등) |

## 4. 주요 기능 (Key Features)

### 4.1 회원 기능

#### 4.1.1 회원가입 및 인증
- **기능 설명**: 이메일 또는 전화번호 기반 회원가입
- **세부 기능**:
  - 회원 유형 선택 (일반회원/트레이너/헬스장 운영자)
  - 이메일 인증 (향후 구현 예정)
  - BCrypt 비밀번호 암호화
  - 프로필 사진 업로드

#### 4.1.2 회원 대시보드
- **기능 설명**: 회원의 헬스장 이용 현황 한눈에 조회
- **세부 기능**:
  - 출결 현황 (이번 달 출석 수)
  - 회원권 정보 (잔여 기간)
  - PT 일정 요약
  - 최근 인바디 기록
  - 운동 목표 관리
  - 헬스장 혼잡도 실시간 조회

#### 4.1.3 회원권 및 상품 관리
- **기능 설명**: 회원권, PT 이용권, 락커 이용권 구매 및 관리
- **세부 기능**:
  - 상품 목록 조회 및 필터링
  - 온라인 결제 (향후 구현 예정)
  - 구매 내역 조회
  - 만료 예정 알림 (향후 구현 예정)

#### 4.1.4 PT 예약 시스템
- **기능 설명**: 트레이너별 PT 예약 및 일정 관리
- **세부 기능**:
  - 트레이너별 가능 시간대 조회
  - 예약 신청 및 취소
  - 예약 승인 알림 (향후 구현 예정)
  - PT 일정 캘린더 뷰

#### 4.1.5 출결 관리
- **기능 설명**: 입실/퇴실 기록 및 출석 이력 조회
- **세부 기능**:
  - QR 코드 또는 전화번호 입력 출결 체크
  - 월별 출석 현황 조회
  - 출석률 통계

#### 4.1.6 인바디 기록 관리
- **기능 설명**: 인바디 측정 기록 등록 및 조회
- **세부 기능**:
  - 인바디 데이터 입력 (체중, 골격근량, 체지방률 등)
  - 기록 목록 조회 및 그래프 시각화 (향후 구현 예정)
  - 최신 인바디 기록 대시보드 표시

#### 4.1.7 운동 목표 관리
- **기능 설명**: 개인 운동 목표 설정 및 달성 여부 관리
- **세부 기능**:
  - 목표 추가/삭제
  - 목표 달성 상태 토글 (미달성/달성)
  - 달성한 목표 일괄 삭제

#### 4.1.8 헬스장 정보 조회
- **기능 설명**: 헬스장 검색 및 상세 정보 조회
- **세부 기능**:
  - 지역별/이름별 헬스장 검색
  - 헬스장 상세 정보 모달 (운영시간, 시설 정보)
  - 혼잡도 조회 (실시간 이용자 수)
  - 기구 목록 조회

#### 4.1.9 방문 예약
- **기능 설명**: 헬스장 방문 예약 신청
- **세부 기능**:
  - 방문 희망 날짜 및 시간 선택
  - 예약 상태 조회 (대기중/승인됨/거절됨)

#### 4.1.10 운동 영상 시청
- **기능 설명**: 헬스장에서 제공하는 운동 영상 시청
- **세부 기능**:
  - YouTube 임베드 영상 재생

## 5. 기능 요구사항 (Functional Requirements)

### 5.1 인증 및 권한 관리

#### FR-1: 회원가입
- **우선순위**: 필수
- **설명**: 사용자는 이메일 또는 전화번호로 회원가입할 수 있어야 한다.
- **상세 요구사항**:
  - 회원 유형 선택 (일반회원/트레이너/헬스장 운영자)
  - 필수 입력 항목: 이름, 이메일, 전화번호, 비밀번호
  - 비밀번호는 BCrypt로 암호화하여 저장
  - 중복 이메일/전화번호 검증

#### FR-2: 로그인
- **우선순위**: 필수
- **설명**: 사용자는 이메일과 비밀번호로 로그인할 수 있어야 한다.
- **상세 요구사항**:
  - 세션 기반 인증
  - 로그인 실패 시 에러 메시지 표시
  - 로그인 성공 시 역할별 대시보드로 리다이렉트

#### FR-3: 역할 기반 접근 제어
- **우선순위**: 필수
- **설명**: 사용자 역할에 따라 접근 가능한 페이지가 제한되어야 한다.
- **상세 요구사항**:
  - 일반회원: 회원 전용 페이지만 접근
  - 트레이너: 트레이너 전용 페이지 + 회원 전용 페이지 접근
  - 헬스장 운영자: 모든 관리 페이지 접근

### 5.2 회원 관리

#### FR-4: 회원 정보 조회 및 수정
- **우선순위**: 필수
- **설명**: 사용자는 자신의 정보를 조회하고 수정할 수 있어야 한다.
- **상세 요구사항**:
  - 프로필 정보 조회
  - 프로필 정보 수정 (이름, 전화번호, 프로필 사진)

#### FR-5: 회원권 관리

- **상세 요구사항**:
  - 회원권 등록 (기간, 가격, 상품명)
  - 회원권 수정
  - 회원권 삭제
  - 회원권 만료 시 자동으로 gym_no NULL 처리

### 5.3 PT 예약 시스템

#### FR-7: PT 예약 승인/거절

## 8. 시스템 아키텍처 (System Architecture)

### 8.1 아키텍처 패턴
- **MVC (Model-View-Controller) 패턴**
  - **Model**: VO 클래스, Mapper 인터페이스
  - **View**: JSP 페이지
  - **Controller**: Spring Controller

### 8.2 계층 구조
```
┌─────────────────────────────────────────────────┐
│               Presentation Layer                 │
│        (JSP, JSTL, JavaScript, CSS)             │
└─────────────────────────────────────────────────┘
                      ▼
┌─────────────────────────────────────────────────┐
│               Controller Layer                   │
│          (Spring MVC Controllers)                │
└─────────────────────────────────────────────────┘
                      ▼
┌─────────────────────────────────────────────────┐
│               Service Layer                      │
│    (Business Logic, Service Interfaces)         │
└─────────────────────────────────────────────────┘
                      ▼
┌─────────────────────────────────────────────────┐
│               Data Access Layer                  │
│       (MyBatis Mapper, Mapper XML)              │
└─────────────────────────────────────────────────┘
                      ▼
┌─────────────────────────────────────────────────┐
│               Database Layer                     │
│              (Oracle Database)                   │
└─────────────────────────────────────────────────┘
```

### 8.3 데이터베이스 설계
- **주요 테이블**: MEMBER, GYM, MEMBERSHIP, PT_RESERVE, ATTENDANCE, STOCK, SALES 등

- **목표**: 제휴 헬스장 평균 매출 20% 증가

- **목표**: 0.1% 이하

## 14. 용어 정의 (Glossary)

| 용어 | 정의 |
|------|------|
| **회원권** | 헬스장 이용 권한이 포함된 상품 (기간제) |
| **PT 이용권** | 퍼스널 트레이닝 이용 권한 (횟수제) |
| **락커 이용권** | 락커 사용 권한 (기간제) |
| **인바디** | 체성분 분석 기록 (체중, 골격근량, 체지방률 등) |
| **출결** | 입실/퇴실 기록 |
| **혼잡도** | 현재 헬스장 이용자 수 (입실만 있고 퇴실 없는 인원) |
| **방문 예약** | 헬스장 첫 방문 또는 상담 예약 |
| **PT 예약** | 트레이너와의 PT 일정 예약 |
| **만료 예정 회원** | 7일 이내에 회원권이 만료되는 회원 |
| **gym_no** | 회원이 소속된 헬스장 번호 (NULL 가능) |
